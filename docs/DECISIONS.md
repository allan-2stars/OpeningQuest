# Opening Quest - Decisions

## D-001

Status:
Approved

Decision:
Opening Quest is Web PWA first.

Reason:
Single codebase, fast testing, tablet friendly.

## D-002

Status:
Approved

Decision:
Tablet-first UX.

Reason:
Chess board and adventure map fit tablets best.

## D-003

Status:
Approved

Decision:
Offline-first V1.

Reason:
Simpler architecture and no backend dependency.

## D-004

Status:
Approved

Decision:
No backend in V1.

Reason:
Core training can work locally.

## D-005

Status:
Approved

React + Vite + TypeScript.

## D-006

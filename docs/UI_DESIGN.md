# Opening Quest - UI Design System

## Design Goals

Opening Quest should feel:
- Friendly
- Motivating
- Colorful
- Professional
- Educational
- Safe for children

References:
- Duolingo style progression
- Brilliant clarity
- Plants vs Zombies style world map energy

## Platform Priority

1. Tablet
2. Desktop browser
3. Mobile phone

The app is tablet-first.

## Visual Style

Cartoon 2.5D.

Avoid:
- Pixel art
- Realistic chess art
- Anime-heavy style
- Dark serious chess-club look

## Color Direction

Primary:
- Royal blue

Secondary:
- Golden yellow

Success:
- Bright green

Warning:
- Orange

Error:
- Soft red

Background:
- Soft gradients
- World-themed palettes

## Mascot

Name:
Hoofy

Concept:
Cute chess-knight creature.

Personality:
- Friendly
- Encouraging
- Short messages
- Never sarcastic
- Never annoying

## Worlds

World 1:

Theme:
Grassland / beginner confidence

World 2:

Theme:

World 3:

## Screens

Required V1 screens:
- Home
- Adventure Map
- Classic Mode
- Practice Board
- Collection
- Profile
- Settings
- Import/Export

## Adventure Map

Use curved path.

Node states:
- Locked
- Available
- Current
- Completed
- Mastered
- Review Due

Boss node must look larger and more special.

## Practice Screen

Top:
- Lesson title
- Progress
- Mastery status

Center:
- Chessboard

Bottom:
- Feedback
- Hint/action area
- Mode indicator

## Feedback

Correct:
- Small bounce
- Soft success sound later

Wrong:
- Gentle shake
- Soft red highlight
- Show correct move in Guided Mode only

## Animation Rules

Use short animations:
- 150ms to 300ms

Avoid:
- flashing
- long transitions
- excessive motion
